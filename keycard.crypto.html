keycard.crypto.ecc module¶

This module provides utility functions for Elliptic Curve Cryptography (ECC) operations using the PyCryptodome library.

keycard.crypto.ecc.derive_shared_secret(private_key, peer_public_key)¶

Derives a shared secret using Elliptic Curve Diffie-Hellman (ECDH) key exchange.

Return type:

bytes

Parameters:

  • private_key (ECC.EccKey) – The local party’s private ECC key.

  • peer_public_key (ECC.EccKey) – The peer’s public ECC key.

Returns:

The derived shared secret as a 32-byte big-endian value.

Return type:

bytes

Note

This function performs the ECDH operation manually by multiplying the private scalar with the peer’s public point and extracting the x-coordinate as the shared secret.

keycard.crypto.padding module¶

Provides ISO/IEC 9797-1 Padding Method 2 (ISO9797 M2) functions for block cipher operations.

keycard.crypto.padding.iso9797_m2_pad(data, block_size=16)¶

Pads the input data according to ISO/IEC 9797-1 Padding Method 2 (also known as ISO/IEC 9797-1 Padding Method 2 or ISO 7816-4 padding).

This method appends a single byte with the value 0x80 to the data, followed by zero or more bytes with the value 0x00, so that the total length of the padded data is a multiple of the block size.

Return type:

bytes

Parameters:

  • data (bytes) – The input data to be padded.

  • block_size (int, optional) – The block size to pad to. Defaults to 16.

Returns:

The padded data.

Return type:

bytes

keycard.crypto.padding.iso9797_m2_unpad(padded)¶

Remove ISO/IEC 9797-1 Padding Method 2 (ISO9797 M2) from the given byte sequence.

This padding method appends a single byte 0x80 followed by zero or more 0x00 bytes. The function locates the last occurrence of 0x80 and removes it along with any trailing 0x00 bytes.

Return type:

bytes

Parameters:

padded (bytes) – The padded byte sequence.

Returns:

The original unpadded byte sequence.

Return type:

bytes

Raises:

ValueError – If the padding is invalid (i.e., 0x80 byte not found).
